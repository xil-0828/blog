# VOD記事作成プロンプトガイド（詳細版）

このガイドは、アニメのVOD配信記事を作るときに、新しい会話で何をプロンプトに書けば良いかを、具体的にまとめたものです。

---

## まず読んでほしいこと

Claudeは会話が終わると記憶がリセットされる。そのため、毎回新しい会話の冒頭で既存の記事を添付する必要がある。これが一番大事なことです。

---

## 会話の冒頭で必ず添付する

既存の記事（このガイドで作った記事であれば、haruhi-vod.md やKanon、Angel Beatsのいずれか1つ）を添付し、以下のように書く。

```
このテンプレートと同じ形で、「[作品名]」のVOD記事をつくって。
配信状況はWebで調べて。
ファイル名は英語にして。
本文はぜっっっっっったいに日本語にしろよ

[記事を添付]
```

それだけで十分になっている。添付した記事のフォーマットを読み取り、同じ構造で新規作品の記事を生成する。

---

## フォーマットの仕様（添付した記事に含まれている）

添付する記事の中には以下の仕様が自動で含まれているため、プロンプトで一つ一つ書く必要はない。

### frontmatter

記事の冒頭にYAML形式のfrontmatterがある。

```yaml
---
title: 'アニメ「[作品名]」の動画配信サービス一覧【2026年】'
summary: '...'
description: '...'
tags: ['...']
pubDate: 'Feb 03 2026'
updatedDate: 'Feb 03 2026'
heroImage: '../../assets/2026-streaming-guide/[作品名].webp'
youtubeId: 'dYIPQPT9KcY'
---
```

heroImageのパスは作品ごとに変える。パスの接頭辞は添付した記事と同じにでよい。youtubeIdは添付した記事と同じ値のままでよい。実際に動画を探す必要はない。

### テーブルのフォーマット

配信状況の一覧テーブルは、以下のように `<img>` タグとCTAリンクで組む。Markdownのリンク記法ではない。

```
| VODサービス | 配信状況 | 無料体験 | 視聴リンク |
| --- | --- | --- | --- |
| <img src="/icons/netflix.png" alt="Netflix" width="120" height="60" /> | 配信なし | なし | — |
| <img src="/icons/amazon-prime.png" alt="Amazon Prime Video" width="120" height="60" /> | レンタル対応 | 30日間 | <a href="https://www.amazon.co.jp/s?k=[検索語]&i=instant-video" class="vod-cta">今すぐ見る</a> |
| <img src="/icons/d-anime.png" alt="dアニメストア" width="120" height="60" /> | 見放題 | 31日間 | <a href="https://animestore.docomo.ne.jp/animestore/sch_pc?searchKey=[検索語]" class="vod-cta">今すぐ見る</a> |
| <img src="/icons/u-next.png" alt="U-NEXT" width="120" height="60" /> | 見放題 | 31日間 | <a href="https://video.unext.jp/search?query=[検索語]" class="vod-cta">今すぐ見る</a> |
| <img src="/icons/dmm-tv.png" alt="DMM TV" width="120" height="60" /> | 見放題 | 14日間 | <a href="https://tv.dmm.com/vod/list/?keyword=[検索語]" class="vod-cta">今すぐ見る</a> |
```

アイコンのパスは固定で、`<a>` のURLの検索語の部分だけ作品ごとに変える。

### 改行の扱い

本文の各行の末尾には2つスペースが入っている。これがMarkdownの改行になっている。添付した記事にこれが含まれているため、生成される記事にも自動で適用される。

---

## 各VODサービスの検索と確認の順番

記事を生成する際には以下の順で検索を行う。

1. 対象作品の配信状況全体を検索する（「[作品名] アニメ 動画配信 見放題」など）
2. Netflixで配信されているか確認する
3. Amazon Prime Videoで見放題かレンタルか確認する
4. dアニメストア / U-NEXT / DMM TVの見放題対応を確認する
5. バージョンが複数ある場合は、対象バージョンを明確にし、FAQで説明する

---

## 記事の構成（添付した記事と同じ順番）

添付した記事には以下の順番で内容が並んでいる。生成する記事も同じ順にする。

1. frontmatter
2. 見出し＋一言まとめ（「〇〇は、○○・○○で見放題配信中です。」）
3. 作品の簡単な説明
4. 目次
5. 配信状況テーブル
6. 各サービスごとの個別セクション（Netflix・Amazon・dアニメストア・U-NEXT・DMM TV）
7. おすすめ
8. FAQ
9. まとめ

## プロンプトの実例

### 新しい会話で最初の記事を作る場合

```
このテンプレートと同じ形で、「けいおん！」のVOD記事をつくって。
配信状況はWebで調べて。
ファイル名は英語にして。
本文はぜっっっっっったいに日本語にしろよ

[haruhi-vod.md を添付]
```

### 同じ会話で続けて別の作品を作る場合

```
次は「Clannad」でつくって
```

これだけで十分。前の記事のフォーマットを引き継いで新規作成する。

### 記事に問題があった場合の修正

```
おかしいから直して
```

何がおかしいかを言わなくても、添付したテンプレートと比較して問題を見つけて修正する。

### 特別な要件がある場合

```
「バカと言葉と」のVOD記事をつくって。
劇場版についてもFAQに書いて。
```

## よくある疑問

### heroImageのパスは毎回変えるべき？

はい。作品ごとに変える。添付した記事のパスの接頭辞を見て、末尾の作品名だけ変える。

添付した記事が `../../assets/2026-streaming-guide/haruhi-suzumiya.webp` なら、けいおんは `../../assets/2026-streaming-guide/keion.webp` になる。

### youtubeIdは毎回違う動画にすべき？

プロンプトで指定しなければ、添付した記事と同じ値のままになる。実際に動画を探してほしい場合は「YouTubeのトレーラーURLも探して」と書く。

### バージョンが複数ある場合は？

記事の冒頭に「本記事では〇〇年版についてまとめています」という注書きが入り、FAQで各バージョンの違いが説明される。これは自動で判断されるが、特定のバージョンを指定したい場合は明示する。

### 新規サービスを追加したい場合は？

「TSUTAYA DISCASも追加で書いて」のように書けば、テーブルと個別セクションに追加される。
